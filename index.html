Scanner


Para poder hacer los ejercicios, necesitamos utilizar peticiones por teclado, más conocidas en Java como Scanner. Los Scanner son un método (más adelante profundizaremos sobre ellos) que permite ingresar por teclado un valor y ese valor asignárselo a una variable. No pertenecen a Java nativo y, para poder utilizarlos, necesitamos importarlos de una librería de métodos. Si te suena a chino tranquila, es normal. Por ahora, solo tienes que saber que debes poner la siguiente línea de código justo debajo del package: import  java.util.Scanner;

Para utilizar un Scanner se requieren 2 pasos:

1. Declarar el Scanner. Esto se hace siguiendo la expresión: Scanner nombreScanner = new Scanner (System.in);

Ejemplo: Scanner teclado = new Scanner (System.in);

El nombre del Scanner lo elegimos nosotros y, al igual que el nombre de las variables, debe ser representativo. Generalmente se utilizan los nombres sc, scanner o teclado, pero si crees uno más adecuado, utilízalo sin problema. La otra parte de la secuencia (new Scanner (System.in)) indica que el valor del método escáner se lo estamos dando introduciendo un valor al sistema. Si te das cuenta, para escribir por pantalla hemos utilizado otro método: System.out.println("");, que indica que estamos "sacando" un valor (System out).

2. Asignar el valor del Scanner a una variable. Es en este paso en el que se permite al usuario escribir por teclado. Para ello, seguimos la sigueinte expresión:

nombreVariable = nombreScanner.nextTipoDeDato( );

Ejemplo: numero = teclado.nextInt( );

Fíjate que acaba en paréntesis vacíos y que el tipo de dato se escribe en UpperCamelCase (inicia con mayúscula). Cuando el tipo de dato es una String (que no es un tipo de dato primitivo) el scanner se asigna como nombreScanner.nextLane( )

Ejemplo: saludo = teclado.nextLane( );

Funciones

La programación estructurada es un paradigma de la programación (un protocolo de reglas que deben seguirse a la hora de estructurar el código) que define las partes de un programa como módulos (o bloques). Imagina un edificio de 10 plantas. Construirlo es difícil si se hacen las diez plantas a la vez, pero si primero se realiza la primera planta, después la segunda y así sucesivamente, el trabajo se facilita. En la programación estructurada es similar, el programa se crea a partir de "plantas" o bloques, cada una constituyendo una parte del "edificio" (el programa).

Las funciones son porciones de código aisladas que realizan una acción concreta y constituyen los módulos (bloques) de un programa. Son especialmente útiles cuando es necesario repetir el mismo bloque de código varias veces a lo largo de un programa. En este simil, sería como si quisiéramos fabricar las puertas de nuestro edificio todas iguales. No tendría sentido diseñar de 0 cada puerta para cada habitación. Es mucho mejor encargar el diseño a una empresa y comprar las puertas que necesitemos cuando las necesitemos. Esto facilita la lectura del código así como su mantenimiento (si queremos modificar el diseño de la puerta, solo tenemos que pedirle a la empresa que lo modifique, no hay que modificar las puertas una por una).

Además se dice que las funciones actúan como "cajas negras", ya que el programa no tiene en cuenta lo que pasa dentro de la función, solo su resultado. En el ejemplo de la construcción del edificio, uno no sabe necesariamente como se construye la puerta, pero obtiene la puerta, que es lo que importa.

Es frecuente escuchar el nombre de métodos cuando se habla de funciones. Son lo mismo estructuralmente, la diferencia es el paradigma de programación en el que se utilizan. En programación estructurada, estos bloques de código se llaman funciones, mientras que en programación orientada a objetos (POO) se denominan métodos.

Las funciones pueden o no retornar un valor cuando se las llama. Un ejemplo de función que no retorna un valor es una en la que quieres mostrar algo por pantalla.

Las funciones pueden ser llamadas por el código principal. Para esto, asignamos la función a una variable de nuestro módulo principal de la siguiente manera:

 
                public static int operacionSumar (int sumando1, int sumando2){
 
                    int suma = sumando1 + sumando2;

                    return suma;

                  }

Vamos a desgranar su sintaxis:

  • public es un modificador de acceso. Le indica a Java qué partes del programa tendrán acceso al funcionamiento interno de la función. Lo veremos a profundidad cuando hablemos de programación orientada a objetos.

  • static es una palabra reservada que indica que estamos ante un método estático. Te habrás quedado igual, de nuevo lo veremos en programación orientada a objetos. Lo único importante aquí es que las funciones que retornan valores llevan static mientras que las que no retornan valores llevan void.

  • int es el tipo de dato que va a enviar la función una vez finalice su ejecución. Si no retorna ningún valor, no se pone el tipo de dato (porque la nada no tiene tipo de dato).

  • operacionSumar es el nombre de la función. Debe estar en infinitivo.

  • (int sumando1, int sumando2) son los parámetros de entrada de la función. Llevan especificados el tipo de dato ya que son las variables con las que trabajará nuestra función. Como son variables exclusivas de la función, se denominan variables locales. Las variables locales solo pueden ser utilizadas en el bloque de código en el que han sido declaradas, y en ningún otro. Los parámetros de entrada reciben una copia del valor de las variables de entrada (las variables de la función principal) y es el valor que tienen durante la ejecución del código de la función. A esta característica se la denomina paso por valor o paso por copia, ya que su valor es una copia del valor de la variable de entrada.

  • return suma indica que la función devuelve un valor, y que este valor es el que está alojado en la variable "suma".


El orden de ejecución del programa es importante para entender cómo actúan las funciones. El programa se ejecuta desde su bloque principal (main) de forma secuencial (línea por línea) hasta que llega a una variable a la que se le ha asignado el nombre de una función:


                public static int operacionSumar (int sumando1, int sumando2){

                    int suma = sumando1 + sumando2;

                    return suma;

                }
                
                public static void main (String[] args){
                     
                    int variable1 = 5;
                    int variable2 = 10;
                    int sumaVariables;

                    System.out.println("Buenos días, me gustaría saber la suma de " + variable1 + " + " + variable2);
                    
                    sumaVariables = operacionSumar(variable1+variable2);
                
                    System.out.println(sumaVariables);

                    System.out.println("Fin de la ejecución del programa");
                }

  • En verde, tendríamos las variables declaradas del módulo principal. Les he asignado un valor manualmente, pero perfectamente puede utilizarse el Scanner para leer un valor por teclado y asignárselo.

  • Tras la declaración de variables, el programa muestra por pantalla el texto "Buenos días, me gustaría saber la suma de 5 + 10".

  • En azul, se observa cómo se asigna la función "operacionSumar" a la variable suma. Para ello, se escriben en los paréntesis las variables que se desean introducir en la función (en este caso para que se sumen), que deben de coincidir en cantidad, y en tipo, para que el programa prosiga. Efectivamente, la cantidad (2 variables separadas por comas) y el tipo de las variables (int) coinciden con los de las función "operacionSumar", por lo que el programa continúa.

  • El programa da un salto y se dirige a la función "operacionSumar". Los valores de las variables "variable1" (5) y "variable2" (10) se copian dentro de los parámetros de entrada "sumando1" y "sumando2", respectivamente. Ahora "sumando1 = 5" y "sumando2 = 10". Se procede a la ejecución del código de la función (suma = 15).

  • Después, la función retorna el valor de la variable "suma" (15) al módulo principal. Esto significa que, donde está el nombre de la función, aparece ahora el valor de la variable "suma". Es decir, ahora "sumaVariables = 15".

  • El programa vuelve al bloque principal y la siguiente instrucción es mostrar por pantalla el valor "15".

  • Por último, se muestra por pantalla "Fin de la ejecución del programa" y el programa termina.


Hay funciones que no retornan ningun valor, simplemente ejecutan una acción. Por ejemplo:

               
            public static void saludar (int veces){

                int indice = 0;

                while(indice<\veces){
                    System.out.println("Hola Jorge :)")
                    indice++;
                }

            }
            
            public static void main (String[] args){
                 
                int saludarVariasVeces;
                int vecesSaludo = 5;
                
                saludarVariasVeces = saludar(vecesSaludo);
            
              System.out.println("Fin de la ejecución del programa");
            }
  • El programa inicia declarando las variables "saludarVariasVeces" y "vecesSaludo". "vecesSaludo" se inicializa, su valor es 5.
  • A la variable "saludarVariasVeces" se le asigna (se llama, se invoca) la función "saludar" y se le indica la variable de entrada "vecesSaludo" que coincide en tipo con el parámetro de entrada de la función.
  • La ejecución del programa salta a la función "saludar", se copia el valor de la variable de entrada en el parámetro de entrada y entra en un bucle que se ejecutará hasta que el indice sea igual al número de veces indicado en el parámetro de entrada "veces" (5). Cada iteración (vuelta) del bucle imprime por pantalla la frase "Hola Jorge :)" e incrementa el valor del índice en 1.
  • Por tanto, lo que se vería por pantalla sería lo siguiente:

    • "Hola Jorge :)" *El valor de la variable índice aumenta en 1 (indice = 0+1 = 1)*

    "Hola Jorge :)" *El valor de la variable índice aumenta en 1 (indice = 1+1 = 2)*

    "Hola Jorge :)" *El valor de la variable índice aumenta en 1 (indice = 2+1 = 3)*

    "Hola Jorge :)" *El valor de la variable índice aumenta en 1 (indice = 3+1 = 4)*

    "Hola Jorge :)" *El valor de la variable índice aumenta en 1 (indice = 4+1 = 5)*. Como 5 no es menor que 5, sale del bucle y vuelve al módulo principal.

  • Por último, se muestra por pantalla "Fin de la ejecución del programa" y el programa termina.
